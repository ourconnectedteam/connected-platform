Google OAuth Consent Screen Configuration

Task 1.3: Update Google Sign-In Consent Screen

Status: Google Sign-In is already configured in Supabase. This guide covers updating the consent screen appearance in Google Cloud Console.


Steps

Step 1: Find Your Google Cloud Project

1. Go to Supabase Dashboard (https://app.supabase.com/) → Your Project → Authentication → Providers → Google
2. Note the Client ID (format: xxxxx.apps.googleusercontent.com)
3. Go to Google Cloud Console (https://console.cloud.google.com/)
4. Select the project that matches your Supabase Google OAuth credentials
5. Navigate to APIs & Services → OAuth consent screen

Step 2: Update App Name

1. Find the "App name" field
2. Change it to: Connected
3. Click "Save"

Step 3: Upload Logo

1. Find the "App logo" section
2. Click "Upload" or "Choose file"
3. Select your Connected logo file
   Requirements: PNG or JPG, 120x120px minimum, max 50KB, square (1:1)
4. Preview to confirm it looks correct
5. Click "Save"

Step 4: Verify Domain (Optional but Recommended)

1. In the OAuth consent screen, find "Authorized domains"
2. First, verify your domain in Google Search Console:
   - Go to Google Search Console (https://search.google.com/search-console)
   - Add your domain property
   - Verify ownership (HTML file, DNS record, etc.)
3. Then add the domain:
   - Return to OAuth consent screen
   - Add your production domain (e.g., connected.com)
   - Click "Save"

Step 5: Save Changes

1. Click "Save and Continue"
2. Review the summary
3. Wait 2-5 minutes for changes to propagate
